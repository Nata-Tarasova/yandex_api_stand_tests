# Автоматизация тестирования API «Яндекс Прилавок»: Создание наборов (Kits) 🛒

Репозиторий содержит пакет автоматизированных тест-кейсов для верификации API-метода создания клиентских наборов (`POST /api/v1/kits`). Все проверки спроектированы на основе требований к валидации поля `name` в теле запроса.

---

## 🛠 Технологический стек
* **Язык программирования:** Python 3.12
* **Тестовый фреймворк:** `pytest`
* **HTTP-запросы:** библиотека `requests`

---

## 🎯 Автоматизированные сценарии (11 тест-кейсов)

В соответствии с чек-листом реализованы следующие автоматические проверки:

### Позитивные проверки (Ожидаемый ответ: `201 Created` + валидация имени):
1. **Граничные значения:** Минимальная длина имени — 1 символ (`test_create_kit_1_letter...`).
2. **Граничные значения:** Максимальная длина имени — 511 символов (`test_create_kit_511_letter...`).
3. **Алфавиты:** Разрешены английские буквы (`"QWErty"`).
4. **Алфавиты:** Разрешены русские буквы (`"Мария"`).
5. **Символы:** Допустимо использование спецсимволов (`"\"№%@\","`).
6. **Символы:** Разрешено использование пробелов (`" Человек и КО "`).
7. **Символы:** Разрешено использование цифр (`"123"`).

### Негативные проверки (Ожидаемый ответ: `400 Bad Request`):
8. **Граничные значения:** Пустое имя / 0 символов (`test_create_kit_empty_name...`).
9. **Граничные значения:** Длина имени выше лимита — 512 символов (`test_create_kit_512_letter...`).
10. **Пропуск параметра:** Тело запроса передается без обязательного поля `name` (`test_create_kit_no_name...`).
11. **Типы данных:** Передача некорректного типа параметра (целое число `123` вместо строки).

---

## 💡 Особенности технической реализации
* **Изоляция тестов:** Перед каждым отдельным тестом динамически генерируется свежий токен авторизации пользователя (`get_new_user_token()`). Это исключает зависимость тестов друг от друга.
* **Безопасность данных:** Модификация тела запроса происходит через метод `.copy()` словаря. Исходный эталонный шаблон данных в модуле `data` гарантированно защищен от перезаписи.
* **Оптимизация кода:** Логика проверок вынесена во вспомогательные функции утверждений (`positive_assert` и `negative_assert_code_400`), что снижает дублирование кода и упрощает его поддержку.

## 🚀 Инструкция по локальному запуску

1. **Клонируйте проект на локальную машину:**
   ```bash
   git clone https://github.com
   cd yandex_api_stand_tests
   ```

2. **Установите необходимые библиотеки:**
   ```bash
   pip install requests pytest
   ```

3. **Запустите тесты с подробным выводом логов в консоль:**
   ```bash
   pytest create_kit_name_kit_test.py -v -s
   ```
